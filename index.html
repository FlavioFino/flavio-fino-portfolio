01 — About

Why I build software

I'm Flavio, a Software Engineering student at the University of Victoria heading into my third year, originally from Italy. My path into coding started with an unlikely spark: watching Sword Art Online as a kid, and being captivated by the idea of a fully immersive VR world. My dream is to one day help build something like that for real — minus the "trapped inside" part.

Since starting my degree, I've learned that software engineering is a much bigger world than just game development — from systems programming to testing to hardware integration — and I've enjoyed exploring all of it. But that original spark still drives me, and game/VR development remains where I ultimately want to end up.

Outside the classroom, I speak Italian, English, Spanish, and some Mandarin — useful more often than you'd think, whether in group projects or just meeting people from different backgrounds.

02 — Projects

What I've built

Smart Climate & Lighting System

Jan – Mar 2026
University of Victoria · Design course · Team of 3 (Software / Mechanical Eng / Electrical Eng)

A cross-disciplinary project tackling energy waste on the UVic campus — classrooms and labs that overheat in summer (where consistent temperature matters, e.g. chemistry labs), and lighting that runs regardless of available natural daylight.

  • Built on a Raspberry Pi Pico with two light sensors and one temperature sensor.
  • Wrote the Python logic to average and compare sensor readings against set thresholds, simulating automated HVAC response (green/yellow LED indicators) and lighting need (a 3-LED scale from no artificial light needed to full output in darkness).
  • Coordinated the software logic with the electrical (wiring/hardware) and mechanical (interface) sides of the team.
Python Raspberry Pi Pico Sensor Integration HTML

WECS Weather App

Mar 2026
University of Victoria · WECS Weather App Workshop

A serverless weather application built while following a hands-on workshop covering serverless functions and real-world third-party API integration.

  • Search-based current conditions, 5-day forecast, and air quality index for any city worldwide.
  • Backend runs as Vercel serverless functions exposing three endpoints, fetching and formatting live JSON data from the OpenWeatherMap API.
  • Third-party API credentials secured via server-side environment variables.
Node.js OpenWeatherMap API Vercel Serverless Vanilla JS
